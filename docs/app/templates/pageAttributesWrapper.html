Framework and Attributes

The framework for the New Zealand Tsunami Database (Database) is based on a series of key data attributes which help to clarify the nature of the published and unpublished data, including records and source events. These attributes are defined below.

In addition to the attributes for records and source events, a section on auxiliary attributes is included at the end of this page for attributes providing additional information on their related primary attributes.

<%= t.pages.attributes.records %>

<%= records %>

<%= t.pages.attributes.sources %>

<%= sources %>

Auxiliary attributes

For some attributes additional information is stored in auxiliary columns that help with their interpretation. While the individual database column names are given below for each attribute, here a quick overview of the different auxiliary columns and possible values

Specificity

Indicating the relevant specificity of related date/time attributes

<%= t.pages.attributes.values %>
Database value Label Description
y Year ignore month, day & time
m Month ignore day & time
d Day ignore time
t Time fully specified

Certainty

Indicating certainty of related quantitative or categorical attributes

<%= t.pages.attributes.values %>
Database value Label
1 Good
2 Questionable/uncertain/range

Comment/details

Additional information, often storing values "as entered"
